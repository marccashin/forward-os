# FORWARD OS — Context Summary **Last updated:** April 23, 2026 | Commit: 93925704 ## Project Overview FORWARD OS is a Vue 3 single-page application for real estate agents, deployed at forward-os.netlify.app. The entire application (approx 1,296,503 chars) exists in a single index.html file without a build step. ## Key Infrastructure - Hosting: Netlify (auto-deploys from GitHub main branch) - Database: Supabase (forward-marketing-os project) - AI: Anthropic Claude API (Sonnet 4.6 default; Opus 4.6 / Haiku 4.5 selectable in Settings) - CRM: Follow Up Boss (Meeting Prep search, Contact Add) - Maps: Google Maps APIs via Netlify function get-nearby-places.js - Email: Resend (agent_feedback daily digest at 6pm ET) - Source Control: GitHub (marccashin/forward-os, auto-deploy enabled) - Backend: Netlify Functions - Video Upload: /video-upload.html — backend NOT YET configured ## Critical Technical Rules - Never double-wrap window.fetch - Use substring splice for patches, never .replace() - Always fetch source from raw.githubusercontent.com before pushing - Verify content length after push - property_notes uses updated_by (not created_by) - If fetch breaks, open a fresh browser tab - Browser filter blocks large string extraction with API keys — use char codes or async chain - All global features must use hardcoded string literals, never runtime window vars ## Netlify Environment Variables - ANTHROPIC_API_KEY - GOOGLE_MAPS_PLACES_KEY (Geocoding + Places + Distance Matrix) - FOLLOW_UP_BOSS_API_KEY - GOOGLE_SERVICE_ACCOUNT_KEY (Video Upload — NOT SET) - GOOGLE_DRIVE_FOLDER_ID (Video Upload — NOT SET) ## GitHub App Auth - App ID: 3382390 - Push via: /.netlify/functions/github-push -
